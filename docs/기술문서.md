# 기술 문서 (개발자/관리자용)

> 일반 사용법은 [README](../README.md) 참고. 이 문서는 내부 동작·구조·주의사항을 담는다.

## 동작 방식
- **이미지 엔진(기본 `codex`)**: `~/.codex/auth.json` 의 ChatGPT OAuth 토큰을 읽어
  `chatgpt.com/backend-api/codex/responses` 의 `image_generation` 툴을 호출한다. **API 키 불필요**,
  사용량은 ChatGPT 구독 할당량에서 차감. 모델은 추론 모델(gpt-5.5)이며 이미지는 도구로 생성된다.
- **대체 엔진(`gemini`)**: Google AI Studio 무료 키로 `gemini-2.5-flash-image` 호출(설정에서 전환).
  헤더 `x-goog-api-key`. 키는 코드에 하드코딩하지 않고 설정 화면/환경변수로 주입.
- 토큰 만료 시 `auth.openai.com/oauth/token` 으로 자동 갱신, 실패하면 `codex login` 재로그인.

## 인증 (OAuth)
- `codex login`(Sign in with ChatGPT)이 곧 ChatGPT OAuth 2.0 흐름이다. 토큰은 각 PC의
  `~/.codex/auth.json` 에 저장 → **PC마다 1회 로그인 필요**(로컬 프로그램의 필수 구조).
- 같은 ChatGPT 계정으로 여러 PC에 로그인하면 이미지 생성 할당량은 **계정 단위로 공유(합산)**.
- 로그인이 번거로우면 기존 PC의 `%USERPROFILE%\.codex\auth.json` 을 새 PC 같은 위치로 복사해도 된다.

## 데이터 (모두 로컬 보관)
- `data/app.db` (SQLite): 이미지 메타 + **프롬프트** + 수정 지시문 + 버전 계보
  - `images`(prompt/engine/model/size/file_path/parent_id/kind…), `edit_events`(type/instruction/…), `projects`
- `data/images/*.png`: 생성·수정·병합 결과 이미지
- `data/settings.json`: 엔진/기본옵션/Gemini 키
- 위 3가지는 `.gitignore` 처리 — 저장소에 올라가지 않음.

## 수동 실행 (개발자)
```
pip install -r requirements.txt
python app.py            # http://127.0.0.1:8765/ 자동 오픈
```
- `app.py` 는 포트가 이미 사용 중이면(=이미 실행 중) 새로 띄우지 않고 브라우저만 열고 종료한다.
- Windows `.js` MIME 문제를 피하려 시작 시 MIME 재등록.

## 기술 스택
Python FastAPI + SQLite(SQLAlchemy) + 정적 HTML/CSS/JS(바닐라). 인증: ChatGPT OAuth(codex CLI).

## 주의
- `backend-api/codex/responses` 는 Codex CLI 가 쓰는 **비공식 내부 엔드포인트**(변경 가능).
  메커니즘은 오픈소스 `leeguooooo/chatgpt-imagegen` 의 검증된 방식을 차용했다.
- 마스크(영역) 수정은 전용 mask 파라미터가 아니라 "원본 + 강조 합성본 + 지시문"을 함께 보내는
  방식이라 픽셀 단위 정밀 인페인팅은 보장하지 않는다.
